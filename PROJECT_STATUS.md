## Apps (side-nav shell: `sap.tnt.ToolPage` + `sap.m` routing)
1. **Reconciliation Monitor** — date filter → fetches ECC + HANA LIPS delivery items in parallel → 3 tabs (ECC / HANA / Missing Items). A **Master Data** dropdown (Delivery Item is live; Delivery Header / Sales Doc Header / Sales Doc Item show a "coming soon" placeholder). Missing = in ECC but not HANA (key `Delivery_Delivery`+`Item`); multi-select → **Post to HANA**, rows turn green.
2. **Pick Info** — two modes via a dropdown:
   - **Pick Task** (by Pick ID) → `ObjectHeader` with colour-coded status.
   - **Picker Logs** (by User ID + **date**; user optional = **all pickers that day**) → a **User → Pick → Activity** `sap.ui.table.TreeTable` (colour-coded events, summary chips, **Download as Excel**). Animated picker SVG as the empty state.

## Backend — CAP service `ReconcileServices` (OData V4, HANA), base path `/ReconcileServices`
- `getECCDeliveryItems(CreatedOn : Date)` → ECC items (via a **destination** to ECC)
- `getHanaDeliveryItems(DeliveryDate : Date)` → HANA items
- `updateHanaDeliveryItems(items : [...])` (action) → string
- `getPickTaskDetails(ID : String)` → 1 PickTaskHeader
- `getPickUserLogs(EventDate : Date, User_ID : String @Core.OptionalParameter)` → events for that day
- **ECC returns raw S/4 field names**; `ECC_TO_HANA` in `Reconciliation.controller.js` maps them to the canonical HANA names (`Delivery_Delivery`, `Material_Material`, `DeliveryQuantity`, `SalesUnit_UnitCode`, `StorageLocation`, `Plant`, `Item`) and converts `/Date()/`→date, ISO-duration→time.

## Connectivity
- **Preview (`npm start`)**: `ui5.yaml` `fiori-tools-proxy` forwards `/ReconcileServices/*` → **`http://localhost:4004`** (local `cds watch`). Swap that line to `destination: CloudWMReconcilliation` to hit the deployed CAP.
- **Deployed**: `xs-app.json` routes `/ReconcileServices/*` → **`CloudWMReconcilliation`** destination → deployed CAP.
- **BTP**: org `bsx-sysco-sandbox-f20`, space `sandbox-f20`, region `eu10-004`. Deployed CAP route: `bsx-sysco-sandbox-f20-sandbox-f20-cloudwmreconciliation-srv.cfapps.eu10-004.hana.ondemand.com`.

## Deploy (managed approuter + HTML5 repo + Work Zone)
Generated by Fiori tools: `mta.yaml`, `ui5-deploy.yaml`, `xs-security.json`. Scripts: `build:cf` (ui5 build preload + `ui5-task-zipper`), `build:mta` (`mbt build`), `deploy` (`fiori cfDeploy`).
Flow: `mbt build` → `cf deploy mta_archives/*.mtar` → Work Zone (Content Manager: app → group + role → site → publish). The launchpad tile uses the `crossNavigation` inbound in `manifest.json` (semantic object `LIPSReconciliation`, action `display`).

## OPEN ITEMS (the actual remaining work)
1. **Deploy is blocked** — `cf deploy` fails with *"Missing mandatory service instance …-html5-service"*. Cause: the **html5-apps-repo app-host quota** is held by an **orphan** from an earlier attempt, `cloudwmreconcilliationui-html5-srv`, which has a **service key** blocking its deletion. **Fix:**
   ```bash
   cf service-keys cloudwmreconcilliationui-html5-srv
   cf delete-service-key cloudwmreconcilliationui-html5-srv <key-name> -f
   cf delete-service cloudwmreconcilliationui-html5-srv -f
   cf undeploy combluestonexcloudwmreconcilliationui -f --delete-services --delete-service-keys   # if registered
   cf deploy mta_archives/*.mtar -f
   ```
2. **Local reconciliation 502 "Failed to load destination"** — the ECC call needs a destination (auth context). The dev-proxy to raw `localhost:4004` has no auth, so the CAP can't load it. **Pick Info works locally** (HANA, no destination). Reconciliation works when the UI is **served same-origin by the CAP** (copy `dist/` into the CAP's `app/` folder and open via `cds watch`) or via the **deployed** approuter. (If you see `ECONNREFUSED`, `cds watch` simply isn't running.)
3. **Deployed CAP must be (re)deployed** with the renamed plural `/ReconcileServices` service + the pick functions, or the deployed UI 404s on those calls.
